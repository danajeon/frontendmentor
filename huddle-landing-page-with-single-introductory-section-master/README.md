### What I learned

This project was my first time using Sass. 

- Custom variables for provided background images, fonts, and colors, which reduced repetitive typing and made the code easier to update.
- Nesting, which allowed child elements and hover states to be logically grouped together under their parent selectors. 
- Mixins or @include was not used for this project since there wasn't a lot of repeated styling. However, I understand their value for creating reusable, parameterized styles that can save time in larger projects. 

Overall, using Sass for this project has helped me appreciate how it can improve code organization, maintainability, and scalability compared to plain CSS. 

### Continued development

- **CSS Grid and responsive layouts:** Working through page-level layout vs component-level layout helped me strengthen my understanding of how spacing, row heights, and alignment interact.
- **Sass CSS:** I now have a clear understanding of variables, nesting, and mixins. While mixins weren’t necessary for this project, I look forward to applying them in larger projects with more repeated styling patterns.
- **Refactoring & maintainability:** Even small projects benefit from organized, modular CSS. This project reinforced the value of structuring styles thoughtfully, which I’ll continue prioritizing in future work.

Overall, this project was a great opportunity to strengthen foundational front-end skills, helping me approach more complex projects with confidence and clarity.
